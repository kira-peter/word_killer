# Change: Improve Rhythm Dance Multi-Word Display

## Why
当前节奏舞蹈模式一次只显示一个单词，玩家完成后立即切换到下一个单词。这种设计存在以下问题：
1. **缺乏前瞻性**：玩家无法提前看到即将出现的单词，难以做好心理准备
2. **节奏感不足**：单词切换过于突然，缺少连续性和流畅感
3. **视觉单调**：界面只有一个单词，视觉冲击力不足

通过引入**滑动窗口式的多单词队列**，可以：
- 提供更好的视觉预览，让玩家提前看到接下来要输入的单词
- 创造类似"音游下落式"的节奏感，增强游戏体验
- 保持强制顺序输入的机制，确保难度和公平性
- 通过"完成→移除→补充"的循环，提供连贯的游戏流

## What Changes
- **单词队列系统**：维护固定5个单词的滑动窗口，当前单词在中间位置
  - 队列包含：2个历史单词（已完成）+ 1个当前单词 + 2个预览单词（待输入）
  - 初始状态：前2个位置为空（历史占位符），后3个位置填充随机单词
  - 完成当前单词后，整个队列上移1位，新单词从底部补充
  - 当前可输入单词始终在队列的中间位置（索引2）

- **UI布局调整**（左右分栏）：
  ```
  左半屏（单词列表）           右半屏（节奏条+特效）
  ========================================
    completed-2 (深灰)
    completed-1 (浅灰)
  ----------------------------------------
  Input: curr__ (白色)  <---> RhythmBar
  ----------------------------------------
      upcoming+1 (浅灰)
      upcoming+2 (深灰)
  ```
  - 左半屏：5个单词垂直排列，靠右对齐
  - 右半屏：节奏条始终在中间高度，与当前单词对齐
  - 颜色渐变：深灰(#444) → 浅灰(#888) → 白色 → 浅灰(#888) → 深灰(#444)
  - 当前单词行最开始显示 "Input: [输入内容]"，无需底部独立输入框

- **输入限制**：强制按顺序输入
  - 只有队列中间位置的单词（索引2）可以被输入
  - 尝试输入其他单词的字母会被拒绝

- **节奏条定位**：
  - 节奏条始终显示在屏幕中间高度（右半屏）
  - 与当前单词（索引2）垂直对齐
  - 队列上移时，节奏条位置固定不动

这是功能改进，不涉及破坏性变更，但会显著改变 UI 布局和交互逻辑。

## Impact
- **新增能力规范**:
  - `word-queue` - 单词队列管理（滑动窗口、补充逻辑）

- **受影响规范**:
  - `rhythm-timing` - 修改当前单词的概念，改为队列中间位置单词（索引2）
  - `ui-rendering` - 大幅调整节奏舞蹈模式的 UI 布局，显示多单词列表，左右分栏设计
  - `input-handling` - 修改输入验证逻辑，只接受队列中间位置单词的输入
  - `game-modes` - 修改 RhythmDanceState 数据结构，添加单词队列和中间索引

- **受影响代码**:
  - `pkg/game/rhythm_dance.go` - 添加单词队列管理逻辑
  - `pkg/ui/rhythm_dance.go` - 重构 UI 布局，渲染多单词列表
  - `cmd/word-killer/main.go` - 更新游戏循环中的单词队列逻辑

- **数据依赖**:
  - 添加配置项：`rhythm_dance_visible_words`（可见单词数量，默认 5）
  - 依赖现有的单词词库（short/medium/long）
