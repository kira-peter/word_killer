# Change: Add Classic Game Mode

## Why
实现 Word Killer 的核心游戏体验。第一版需要提供一个完整可玩的经典模式，作为后续其他玩法的基础。经典模式是最直观的打字练习方式，能够立即为用户提供价值。

## What Changes
- **游戏核心逻辑**: 单词生成、匹配检测、消除机制、游戏状态管理
- **输入处理系统**: 实时键盘事件监听、输入缓冲区管理、匹配算法
- **UI 渲染**: 命令行界面渲染、单词高亮显示、屏幕刷新
- **统计系统**: 实时数据收集和计算（敲击数、正确率、速度等）
- **游戏流程**: 开始 → 进行中 → 完成 → 结果展示

这是全新功能，不涉及破坏性变更。

## Impact
- **新增能力规范**:
  - `game-core` - 游戏核心逻辑
  - `input-handling` - 输入处理
  - `ui-rendering` - UI 渲染
  - `statistics` - 统计数据

- **受影响代码**:
  - 需要创建核心游戏包结构
  - 需要集成键盘事件监听库
  - 需要实现命令行 UI 渲染引擎
  - 需要建立主游戏循环

- **外部依赖**:
  - 键盘事件库（待选型）
  - CLI 渲染库（待选型）
  - 单词词库数据
