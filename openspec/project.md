# Project Context

## Purpose
Word Killer 是一个低调但绝不简单的命令行打字游戏。通过有趣的游戏机制帮助用户提升打字速度和准确度。

**核心目标：**
- 提供流畅的命令行交互体验
- 实现多种游戏玩法（第一版先实现经典模式）
- 准确统计和展示玩家表现数据

## Tech Stack
- **语言**: Go
- **UI**: 命令行界面 (CLI)
- **交互**: 键盘事件监听和实时渲染

## Project Conventions

### Code Style
- 遵循 Go 官方代码规范（gofmt）
- 使用有意义的变量和函数命名
- 保持函数简洁，单一职责
- 公开 API 需要清晰的文档注释

### Architecture Patterns
- **分层架构**:
  - UI 层：命令行渲染和用户输入处理
  - 游戏逻辑层：单词管理、匹配检测、状态更新
  - 数据统计层：性能指标计算和记录
- **事件驱动**: 键盘事件监听和响应
- **状态管理**: 集中管理游戏状态（进行中、已完成等）

### Testing Strategy
- 单元测试覆盖核心游戏逻辑
- 集成测试验证完整游戏流程
- 边界条件测试（空输入、特殊字符等）

### Git Workflow
- 功能分支开发
- 提交信息清晰描述改动内容
- 代码审查后合并到主分支

## Domain Context

### 游戏机制
**经典模式（第一版）**:
1. 屏幕中随机产生一定数量的单词
2. 持续监听键盘事件
3. 用户输入与单词匹配时高亮显示
4. 按下回车键消除完全匹配的单词
5. 所有单词消失即为游戏完成

### 统计指标
- **总敲击数**: 用户按键总次数
- **有效数量**: 匹配单词的按键次数
- **正确数**: 正确输入的字符数
- **单词数量**: 成功消除的单词数
- **字母数量**: 成功输入的字母总数
- **总耗时**: 游戏开始到结束的时间
- **速度指标**:
  - 字母数/秒 (字母速度)
  - 单词数/秒 (单词速度)

### 术语
- **高亮**: 当前输入匹配的单词显示特殊样式
- **消除**: 完全匹配并确认后的单词从屏幕移除
- **有效输入**: 能够匹配到屏幕中单词的按键

## Important Constraints
- 必须在命令行环境运行
- 需要实时响应键盘输入（低延迟）
- 跨平台兼容（Windows、macOS、Linux）
- 性能要求：即使在低配置设备上也能流畅运行

## External Dependencies
- 键盘事件捕获库（如 `github.com/eiannone/keyboard` 或类似）
- 命令行 UI 渲染库（如 `github.com/charmbracelet/bubbletea` 或 `github.com/nsf/termbox-go`）
- 单词词库（可以是内置文件或外部数据源）
